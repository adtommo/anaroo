# Contributing to Anaroo

Thanks for your interest in contributing! Pull requests are welcome.

## How to Contribute

1. Create a branch from `main` (or fork the repo if you don't have push access).
2. Make your changes and add tests if applicable.
3. Ensure `npm run build` and `npm test` pass.
4. Open a pull request with a clear description of what you changed and why.

## Good to Know

- This project is maintained by a single person on a best-effort basis.
- All merges are at the sole discretion of the maintainer.
- There is no guaranteed review timeline -- please be patient.
- Keep PRs focused. Smaller, well-scoped changes are easier to review and more likely to be merged.

## Development Setup

See [DEVELOPMENT.md](DEVELOPMENT.md) for environment setup and project structure.

## Code Style

- TypeScript throughout. Follow the existing patterns in the codebase.
- Use [conventional commits](https://www.conventionalcommits.org/) for commit messages (`feat:`, `fix:`, `chore:`, etc.).
